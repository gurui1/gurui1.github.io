JavaScript 对象和 JSON（JavaScript Object Notation）。以下是它们之间的主要区别：

数据类型：

JavaScript 对象：JavaScript 对象是一种原生的数据类型，键值对，用于在代码中表示复杂的数据结构，可以包含各种数据类型（例如字符串、数字、布尔值、数组、函数等），并支持动态添加和修改属性。

JSON：JSON 是一种数据格式，使用文本表示数据，只支持有限的数据类型，包括字符串、数字、布尔值、数组、对象和 null，不支持函数、undefined 等。

语法：

JavaScript 对象：在 JavaScript 中，对象用大括号 { } 定义，属性和值之间使用冒号: 分隔。

JSON：JSON 使用类似 JavaScript 对象字面量的语法，但要求属性名和字符串值都必须用双引号包裹，而 JavaScript 对象字面量中属性名可以不加引号。

用途：

JavaScript 对象：用于在 JavaScript 代码中表示和操作数据结构，支持方法和动态属性。

JSON：用于数据交换和存储，通常在 Web 开发中用于前后端数据传输，以及配置文件的存储。

解析方式：

JavaScript 对象：直接在代码中编写对象字面量即可创建对象，并且可以通过点表示法或方括号表示法访问对象的属性和方法。

JSON：需要使用 JSON.parse() 方法将 JSON 字符串解析为 JavaScript 对象，以及使用 JSON.stringify() 方法将 JavaScript 对象序列化为 JSON 字符串。

总的来说，JavaScript 对象是一种原生的数据类型，用于在代码中表示数据结构和逻辑关系，而 JSON 是一种数据格式，用于数据的交换和存储。在实际开发中，可以通过将 JavaScript 对象序列化为 JSON 字符串来进行数据交换和通信。

JSON.stringify(): 这个方法用于将 JavaScript 对象转换为 JSON 字符串。语法如下：

const obj = {
name: ‘Alice’, age: 30, sayHello() {
console.log(“Hello!”);
}
};
const jsonStr = JSON.stringify(obj);
console.log(obj);
console.log(jsonStr); ## 输出转换后的 JSON 字符串 {“name”:”Alice”,”age”:30}

obj 对象包含一个方法 sayHello，但当它被转换为 JSON 字符串时，方法会被忽略

JSON.parse(): 这个方法用于将 JSON 字符串解析为 JavaScript 对象

const jsonStr1 = ‘{“name”:”Alice”,”age”:30}’;
const obj1 = JSON.parse(jsonStr1);

console.log(obj1); ## 输出解析后的 JavaScript 对象 { name: ‘Alice’, age: 30 }
