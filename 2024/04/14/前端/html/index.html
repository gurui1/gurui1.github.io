一 元素分类

块级元素 行内元素（内联元素） 内联块元素

1 块级元素 (Block-level elements)：

    一个块级元素自己占一行
1  默认情况下，块级元素会占据其父元素的完整宽度（从左到右），并在其前后形成“断行”（即自动换行）。
2  常见的块级元素包括 <@@div>、<@@p>、<@@h1>-<@@h6>、<@@ul>、<@@ol>、<@@li> 等。
3  块级元素可以包含其他块级元素或行内元素。
4  可以设置宽度和高度。 
<@@div>111<@@/div>
<@@p>222<@@/p>
<@@h1>333<@@/h1>

2 行内元素 (Inline elements)：

    多个行内元素可以在一行显示 直到超出父容器宽度才换行 
1 行内元素不会独占一行，其宽度只与内容的宽度相关。
2 常见的行内元素包括 <@@span>、<@@a>、<@@img>、<@@strong>、<@@em> 等。
3 行内元素主要用于不打断段落中的文本流。
4 通常不可以设置宽度和高度，其大小直接受内容影响。
<@@span>444<@@/span>
<@@a href="">55<@@/a>
<@@img src="../../assets//充值记录.png" alt="" width="40px" height="30px">

3 内联块元素 (Inline-block elements)：

    使得块级元素不换行 行内元素可以设置高度宽度
1 是一种特殊类型，结合了行内元素和块级元素的特性。
2 行为类似于行内元素，即不会断行，但可以设置宽度和高度，类似块级元素。
3 例如，通过 CSS 将元素的 display 属性设置为 inline-block 可以使其表现为内联块元素。 -->

<@@div class="inline-block">Inline-block Element<@@/div>
<@@div class="inline-block">Inline-block Element<@@/div>
<@@span class="inline-block">Another Inline-block Element<@@/span>

三 HTML5 Web 存储

1 localStorage - 用于长久保存整个网站的数据，保存的数据没有过期时间，直到手动去除

    localStorage 的值类型限定为string类型
（在本网站一直存在 本网站的其他小页面里也是这个值 ）
    方法 1 代码清除 localStorage.clear(); 
    方法 2 浏览器 F12 应用 本地存储删除 
存储
localStorage.setItem("sitename", "菜鸟教程");
查找
document.getElementById("result").innerHTML = "网站名：" +  localStorage.getItem("sitename");
清除
localStorage.clear()

2 sessionStorage - 用于临时保存同一窗口(或标签页)的数据，在关闭窗口或标签页之后将会删除这些数据。

（只在当前这个小页面存在 当页面关闭时自动删除 不会影响其他页面 ）
    要向 sessionStorage 中添加数据，可以使用 setItem 方法。此方法接受两个参数：一个是键（key），另一个是值（value）。值必须是字符串。如果需要存储非字符串数据（如对象或数组），可以先将其转换为 JSON 字符串。
    方法 1 代码清除 sessionStorage.clear(); 
    方法 2 浏览器 F12 应用 会话存储删除 
存储
sessionStorage.setItem('userInfo', JSON.stringify({ name: 'John', age: 30 }));
查找
let data = sessionStorage.getItem('key'); // 返回 'value'
let userInfo = JSON.parse(sessionStorage.getItem('userInfo')); // 解析并返回对象
移除数据
sessionStorage.removeItem('key'); // 移除 'key' 对应的数据 
清空所有数据
sessionStorage.clear(); // 清空所有 sessionStorage 数据

四 应用程序缓存

HTML5 引入了应用程序缓存，这意味着 web 应用可进行缓存，并可在没有因特网连接时进行访问。
应用程序缓存为应用带来三个优势：
1 离线浏览 - 用户可在应用离线时使用它们
2 速度 - 已缓存资源加载得更快
3 减少服务器负载 - 浏览器将只从服务器下载更新过或更改过的资源。
