展现vue

数据响应式：数据的变化会引发界面的自动更新（粗犷）

在vue组件里引入css文件夹 以便于达到复用 简洁

import ‘../css/main.css’ 局部引入css文件
<1style src=”../css//main.css”></1style> 全局引入css文件

在vue组件里引入js文件夹 以便于达到复用 简洁

1 引入特定的函数

在js文件夹里面暴露出去

export default function createChildElements(): void {

}
export const xx = ref(1);

import createChildElements from “../ts/ceshi.ts” 引入函数

将原来的代码封装成一个函数，并导出该函数，以便在 Vue 组件中引入并调用。这种封装的方式更加模块化和可维护，也更符合良好的编程实践。

全局引入

<1script>
// 在这里一次性导入整个模块的所有导出内容
import * as utils from “../ts/ceshi.ts”;
//* as utils 的意思是将整个模块作为一个对象导入，并将其命名为 utils。这样做的好处是可以在代码中方便地使用该模块的所有导出内容，而无需逐个引入。
从而整个vue页面都可以用utils里面的数据
// 可以在这里使用导入的模块内容
console.log(utils.xx.value);
</1script>
